# Pinetree Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a static Next.js app that visualizes 宋元學案 scholars as a 3D embedding projection and hierarchical dendrogram, deployed to GitHub Pages.

**Architecture:** Pre-processing pipeline (Gemini agent splits scholars → Gemini Embedding generates vectors → UMAP/t-SNE reduces to 3D → agglomerative clustering builds dendrogram → bundle to JSON). Frontend is a Next.js static export with Two tabs: 3D scatter plot (React Three Fiber) and dendrogram (D3.js), sharing a right panel for scholar details and xuean filters.

**Tech Stack:** Next.js, React Three Fiber, D3.js, shadcn/ui, Tailwind CSS, `@google/genai`, `umap-js`, `ml-hclust`, `gh-pages`

**Spec:** `docs/superpowers/specs/2026-03-26-pinetree-design.md`

- [ ] **Step 2: Create color utility**

Create `src/lib/colors.ts`:
```typescript
/**
 * Generate evenly-spaced HSL colors as hex strings.
 * Works in both Node.js (scripts) and browser (frontend).
 */
export function generateXueanColors(count: number): string[] {
  const colors: string[] = [];
  for (let i = 0; i < count; i++) {
    const hue = (i * 360) / count;
    colors.push(hslToHex(hue, 70, 55));
  }
  return colors;
}

function hslToHex(h: number, s: number, l: number): string {
  s /= 100;
  l /= 100;
  const a = s * Math.min(l, 1 - l);
  const f = (n: number) => {
    const k = (n + h / 30) % 12;
    const color = l - a * Math.max(Math.min(k - 3, 9 - k, 1), -1);
    return Math.round(255 * color)
      .toString(16)
      .padStart(2, "0");
  };
  return `#${f(0)}${f(8)}${f(4)}`;
}
```

- [ ] **Step 3: Create normalize utility for scripts**

Create `scripts/lib/normalize.ts`:
```typescript
const SCALE = 20;

export function normalize3D(
  positions: [number, number, number][]
): [number, number, number][] {
  if (positions.length === 0) return [];

  const mins = [Infinity, Infinity, Infinity];
  const maxs = [-Infinity, -Infinity, -Infinity];

  for (const p of positions) {
    for (let i = 0; i < 3; i++) {
      if (p[i] < mins[i]) mins[i] = p[i];
      if (p[i] > maxs[i]) maxs[i] = p[i];
    }
  }

  const ranges = maxs.map((max, i) => max - mins[i]);
  const maxRange = Math.max(...ranges);

  return positions.map((p) =>
    p.map((v, i) => {
      const center = (maxs[i] + mins[i]) / 2;
      return maxRange === 0 ? 0 : ((v - center) / maxRange) * SCALE * 2;
    }) as [number, number, number]
  );
}
```

The prompt for each volume should be structured as follows (adapt as needed):

```
You are a classical Chinese text parser specializing in 宋元學案.

Given a volume of text, extract each individual scholar entry. Scholar entries typically begin with a header line in the format: [官職/諡號][姓][字/號]先生[名] (e.g., "文昭胡安定先生瑗", "節孝徐仲車先生積").

Sections beginning with ◆ (e.g., "◆安定門人") indicate relationship groupings — use these as the "section" field.

The 學案 name is stated in the volume header (first line), e.g., "安定學案" from "第001卷 卷一 安定學案".

For each scholar, return a JSON object with these fields:
- "name": Chinese name only (e.g., "胡瑗")
- "namePinyin": Pinyin with tone marks removed, capitalized (e.g., "Hu Yuan")
- "courtesy": Courtesy name / 字 (e.g., "翼之"), or "" if not found
- "title": Posthumous/official title (e.g., "文昭"), or "" if not found
- "xuean": The 學案 name from the volume header (e.g., "安定學案")
- "xueanPinyin": Pinyin of the xuean name without "學案" suffix (e.g., "Anding")
- "section": The ◆ section header this scholar appears under (e.g., "安定門人")
- "text": The full text of this scholar's entry, with proper modern Chinese punctuation

Return ONLY a JSON array. No markdown fences, no explanation.
```

Expected output per volume (example):
```json
[
  {
    "name": "胡瑗",
    "namePinyin": "Hu Yuan",
    "courtesy": "翼之",
    "title": "文昭",
    "xuean": "安定學案",
    "xueanPinyin": "Anding",
    "section": "高平講友",
    "text": "胡瑗，字翼之，泰州如人。七歲善屬文..."
```

Key implementation details:
- Process volumes sequentially to respect rate limits
- Skip volume 000 (卷首 — front matter / table of contents, not individual scholar entries)

- Add 1-second delay between API calls
- Log progress (e.g., "Processing volume 5/102...")
- Write intermediate results per volume to `scripts/output/scholars-v{NNN}.json` to allow resuming on failure
- After all volumes are processed, merge into `scripts/output/scholars.json` and assign IDs (`v{volume}-{pinyin-slug}`)

```typescript
"use client";

import { useState, useEffect } from "react";
import type { Dataset } from "@/types";

export function useDataset() {
  const [dataset, setDataset] = useState<Dataset | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const basePath = process.env.NODE_ENV === "production" ? "/pinetree" : "";
    fetch(`${basePath}/data/dataset.json`)
      .then((res) => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then(setDataset)
      .catch((err) => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  return { dataset, loading, error };
}
```

```typescript
"use client";

import { useState, useCallback } from "react";

export function useFilter(xueanIds: string[]) {
  const [hiddenIds, setHiddenIds] = useState<Set<string>>(new Set());

  const toggle = useCallback((id: string) => {
    setHiddenIds((prev) => {
      const next = new Set(prev);
      if (next.has(id)) next.delete(id);
      else next.add(id);
      return next;
    });
  }, []);

  const isVisible = useCallback(
    (xueanId: string) => !hiddenIds.has(xueanId),
    [hiddenIds]
  );

  return { hiddenIds, toggle, isVisible };
}
```

- [ ] **Step 2: Build and deploy**

```bash
pnpm deploy
```

This runs `next build && touch out/.nojekyll && gh-pages -d out -t true`, which:
- Builds the static export to `out/`
- Adds `.nojekyll` to prevent Jekyll processing
- Pushes `out/` to the `gh-pages` branch

- [ ] **Step 3: Enable GitHub Pages**

```bash
gh api repos/dhchoi/pinetree/pages -X POST -f source.branch=gh-pages -f source.path="/" 2>/dev/null || \

```
